# 個人サイト 引継ぎメモ（AI向け）

新しいチャットでこのメモをClaudeに渡せば、経緯を踏まえて続きの作業ができます。
このファイル自体はGitHubにアップロードしない（ローカル保管のみ）。

## プロジェクト概要
- 個人のポートフォリオサイト。小説・映像・イラストなど、**作品ごと**にLP(ランディングページ)を作る構成
  - 以前は「小説/映像/イラスト」というメディア区分でカテゴリ分けしていたが、1作品の中に小説と映像が両方ある場合があるため**作品単位**の構成に変更済み（2026-06-23）
- 公開方法：GitHub Pages（無料）
  - 無料プランは **Publicリポジトリ必須**
  - 公開URL：`https://sashiemichkazu.github.io/`
- 更新方針：ユーザーは自分でコードを書かず、その都度Claudeに依頼して更新する。GitHubはWeb UI（ドラッグ&ドロップ）のみ使用、git/コマンドラインは使わない
  - → **ファイル1個あたり25MBまで**（GitHubのWebアップロード上限）。これを超える場合は圧縮が必要
- 作者名（ペンネーム）：**SashieMichkazu**（日本語表記：サシエ・ミチカズ）。実名ではないので個人情報ルールには抵触しない

## サイト構成（ツリー構造・2026-06-23時点）
```
トップページ (index.html) … 作者名 + 作品グリッド（画像付きカード）
├─ assets/                       … トップページ用のサムネイル画像
└─ works/
    └─ star-wars-evil-paladin/   … 1作目「STAR WARS: EVIL PALADIN」のLP本体
        ├─ index.html            … 元 black_paladin_lp.html。動画src差し替え済み
        ├─ Star_Wars_EP0408_web.mp4  … 圧縮済みヒーロー動画(約16MB, 480x874, 5分12秒)
        └─ 各キャラクター/勢力の画像(圧縮済みPNG/元のままのJPG)
```
今後、新しい作品を追加する場合は `works/作品スラッグ/` を1つ作り、トップページの `.work-grid` にカードを1つ追加する。

旧 novels/ movies/ illustrations/ フォルダは削除済み（ローカル・本番GitHub双方）。

## 作品グリッドのデザイン
- `.work-grid` (CSS Grid, auto-fill, カード幅220px〜) に `.work-card` を並べる
- カード = 画像(3:4, object-fit:cover) + 作品タイトル + タグ行
- 準備中の作品は `.work-card.is-placeholder`（リンクなし、半透明）で表示
- ベースの白背景・明朝体デザインは維持。装飾は最小限
